连接失败时反复更换节点，往往会掩盖真正原因。更有效的方法是按“账户与订阅 → 客户端 → 本机网络 → 节点与服务端”的顺序排查，每一步只改变一个变量。

如果确认原服务已经长期异常，需要比较备用方案，可查看二毛主站持续更新的[2026 机场推荐与风险汇总](https://www.ermao.net/posts/vpn/)。建议先完成本文排查，避免把本机配置问题误判成服务端故障。

## 先判断故障范围

回答四个问题：

1. 所有设备都失败，还是只有一台设备？
2. 所有节点都失败，还是只有某个地区？
3. 切换 Wi-Fi 和移动网络后结果是否相同？
4. 是订阅更新失败、节点连接失败，还是只有某个网站打不开？

| 现象 | 更可能的方向 |
|---|---|
| 所有设备、所有网络都失败 | 账户、订阅或服务端异常 |
| 只有一台设备失败 | 客户端、本机代理或权限问题 |
| Wi-Fi 失败、移动网络正常 | 路由器、DNS 或当前网络问题 |
| 节点能连但单个网站失败 | 规则、DNS、地区限制或目标网站故障 |

## 第一步：检查账户和订阅

- 登录官网确认套餐没有到期、流量没有用完。
- 查看公告是否正在维护，订阅地址是否被更换。
- 在客户端执行“更新订阅”，记录完整错误信息。
- 如果订阅返回空内容或网页错误，不要反复删除客户端配置，先联系服务商。

不要把订阅链接发到公开群组或在线转换网站。它通常包含账户凭据，泄露后可能被他人使用。

## 第二步：检查客户端状态

1. 确认系统代理或 VPN 权限已经开启。
2. 检查当前选中的配置文件和节点。
3. 临时切换到全局模式测试；如果全局可用而规则模式失败，问题多半在规则或 DNS。
4. 关闭其他 VPN、代理、广告过滤和安全软件后复测，避免多个网络扩展冲突。
5. 核对系统日期、时间和时区。时间偏差可能导致证书连接失败。

完成测试后恢复原来的规则模式。全局模式适合定位问题，不一定适合长期使用。

## 第三步：做最小化网络测试

- 从当前 Wi-Fi 切换到移动热点；
- 关闭路由器上的代理插件，只测试单个客户端；
- 选择一个服务商明确标注为可用的普通节点；
- 先测试普通网页，再测试视频、AI 或地区限定服务。

如果换网络后立即恢复，重点检查路由器 DNS、IPv6、访客网络限制和上游网络。不要同时修改五六个选项，否则无法知道是哪项生效。

## 第四步：区分 DNS 与节点问题

典型 DNS 问题包括：节点显示已连接，但域名打不开；部分网站跳转到错误地址；规则模式失败而全局模式正常。

这时先恢复客户端默认 DNS 配置并重启网络。不要随意复制来源不明的复杂 DNS 模板。若错误日志出现 `timeout`、`certificate`、`authentication` 或 `no such host`，提交工单时应一并附上，但需要遮住订阅链接、UUID、邮箱和 IP 等敏感信息。

## 什么时候可以判断是机场故障

满足以下条件时，服务端异常的可能性较高：

- 账户和流量正常；
- 多台设备、多个网络结果一致；
- 重新获取订阅后全部节点仍失败；
- 官方群组出现同时间、同地区的大量报告；
- 状态页或公告确认维护。

这时继续重装客户端意义不大。保存错误时间、网络运营商、客户端版本和两个代表节点的日志，提交工单并等待公告。

## 不建议做的操作

- 下载群聊中来源不明的“修复版”客户端；
- 把完整订阅链接或配置截图公开发送；
- 为排障关闭系统全部安全保护；
- 未确认原因就购买另一个长期套餐；
- 将一次连接失败直接等同于跑路。

若官网、节点和客服同时长期失联，再结合付款是否仍开放、公告是否停止更新，参照本站的[机场风险记录](/tags/机场跑路/)进行判断。

## 排查完成后的选择

- 属于客户端或本地网络问题：保留当前套餐，记录有效配置。
- 属于临时维护：关注官方公告，不要重复购买同一运营主体的套餐。
- 属于长期服务异常：先阅读[机场购买前 12 项检查表](/posts/jichang-buying-checklist/)，再到[二毛主站机场推荐总表](https://www.ermao.net/posts/vpn/)选择不同运营主体的小周期方案。
